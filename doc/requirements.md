# Requirements

## 1. Overview

《Boing 声音软盒》是一款移动端采样声音玩具：玩家录制 0.4–4 秒声音，录音被切成 7 颗弹性声音体，随透明盒子的旋转与碰撞生成节奏和音高变化。

## 2. Visual Design

录音入口是一枚悬浮的透明录音核心；完成后核心连续形变为一只平衡在角上的半透明软盒。盒内有 7 颗蓝黄弹性体，碰撞时压扁、发光并推起盒壁；盒面带点阵、实时声谱短柱和能量刻度。整体采用冷灰蓝摄影棚背景、玻璃高光和极少量深色文字，不沿用植物图鉴的平面线描结构。

## 3. Game Mechanics

- 录音时长为 0.4–4 秒；松手结束，4 秒自动结束。没有麦克风权限时可使用 2.4 秒合成示例。
- 录音完成后自动生成 7 个声音体并立刻进入播放状态，不要求额外选择效果。
- 录音解码后按峰值与 RMS 做最多 8 倍安全响度归一化，再分成 7 段搜索各段 90 ms 能量最高点；每个声音体持有一个固定热点，撞墙或互相碰撞时触发 110–320 ms 的采样颗粒，而不是连续并行播放整段录音。
- 碰撞音高量化到五声音阶 `0 / 2 / 4 / 7 / 9 / 12 / 14` 半音；同一声音体在 72 ms 内不重复触发，整体最多同时保留 10 个颗粒声部。
- 物理循环使用 `requestAnimationFrame`，7 个声音体受盒体方向变化后的重力、阻尼和互斥力影响；撞墙时速度按弹性系数反射。
- 三个参数范围均为 0–100：弹性控制反射系数 `0.48–0.94` 与碰撞密度，音高控制颗粒转调范围 `±2–14` 半音，空间控制反馈延迟 `70–310 ms` 与混响发送量 `0.03–0.26`。
- 拖动盒子改变 X/Y 姿态；松手后保留惯性并在 8–18 秒内缓慢回到展示角度。
- 发布作品时保持软盒运行，从最终音频总线录制约 5 秒碰撞成品并上传；原始麦克风录音不上传。每位用户最多保留 20 件作品，作品墙显示最近 24 件。
- 社交数据包含作品、按作品 ID 去重的点赞和最长 140 字留言；所有写入先在本地即时回显，再异步同步 Aigram。

## 4. Controls

- 录音入口主按钮使用“按住录音、松开完成”：Pointer 按下开始，Pointer 松开或取消结束；示例按钮直接进入软盒。
- 系统权限弹窗只在用户按住录音后触发；若首次授权打断按压，本次只完成授权并提示再次按住，不生成空录音。入口提前说明录音仅在设备本地处理，等待授权、用户拒绝和容器阻止状态都提供明确反馈。
- 在软盒舞台拖动以旋转盒体并改变重力方向；轻点盒体注入一次冲量。
- 底部播放按钮暂停/继续物理与声音；三个滑块实时改变弹性、音高与空间。
- “重新录音”动态收起软盒并回到录音入口。
- 若嵌入式 Mini App 的 iframe 未开放麦克风权限，入口显示宿主配置诊断并保留示例入口。
- 录音入口顶部和软盒控制条都可进入“声音标本盒”作品墙；作品卡支持播放、作者主页、点赞和留言。
- 发布前展示隐私确认；点赞与留言只在 Aigram 内启用，并通知作品作者。

## 5. Win / Lose Conditions

无胜负和失败。首次录音完成即视为作品生成；暂停后保留当前盒体姿态与三个参数，继续播放时从该状态恢复。

## 6. Sound Effects

- 录音开始：520 Hz 正弦波，70 ms，峰值 0.045。
- 录音完成：330→660 Hz 正弦波滑音，160 ms，峰值 0.055。
- 撞墙：归一化录音热点颗粒，110–320 ms，8 ms 起音与指数衰减，干声增益 `0.66–0.94`，按碰撞速度缩放。
- 声音体互撞：两个对应热点颗粒分别触发，并各自叠加五声音阶短正弦共鸣，150–350 ms；最终经 `-18 dB` 阈值、`3.5:1` 比率的压缩器限制峰值。
- 主输出经过 80 Hz 高通、6.5 kHz 低通、轻量压缩、反馈延迟与 0.9 秒卷积混响，最大总增益 0.78。
